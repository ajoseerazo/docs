---
title: "Payout actualizado"
description: "Se envía cuando un payout cambia de estado."
---

**Tipo de evento:** `payout.update`

El `type` del evento es siempre `payout.update`, sin importar la transición — lee `data.state` (y `data.previousState`) para saber qué pasó. Ver [Resumen de webhooks](/es/api-reference/webhooks/overview) para entrega, headers y verificación de firma.

### Campos de `data`

<ResponseField name="payoutId" type="string">El identificador del payout.</ResponseField>
<ResponseField name="companyId" type="string">El identificador de tu empresa.</ResponseField>
<ResponseField name="externalId" type="string">Tu referencia externa, si se proporcionó.</ResponseField>
<ResponseField name="onBehalfOf" type="string">El usuario/entidad para quien se hizo el payout.</ResponseField>
<ResponseField name="previousState" type="string | null">El estado previo a esta transición. Es `null` en el primer evento.</ResponseField>
<ResponseField name="state" type="string">El estado actual del payout. Ver la tabla de abajo.</ResponseField>
<ResponseField name="amount" type="number">El monto del payout, en unidades mínimas (por ejemplo, centavos).</ResponseField>
<ResponseField name="currency" type="string">La moneda de destino.</ResponseField>
<ResponseField name="destinationType" type="string">Uno de `wallet`, `bank` o `qr_code`.</ResponseField>
<ResponseField name="errorReason" type="string">Presente solo cuando el payout falló.</ResponseField>
<ResponseField name="txHash" type="string">Hash de la transacción on-chain. Presente para destinos `wallet` y `bank`.</ResponseField>
<ResponseField name="extraData" type="object">
  Metadatos del riel bancario. Presente solo para `destinationType = bank`.
  <Expandable title="extraData">
    <ResponseField name="finalAmount" type="string">Monto final liquidado.</ResponseField>
    <ResponseField name="finalAmountCurrency" type="string">Moneda del monto liquidado.</ResponseField>
    <ResponseField name="uetr" type="string">Referencia única de transacción de extremo a extremo (SEPA/SWIFT).</ResponseField>
    <ResponseField name="trackingNumber" type="string">Número de seguimiento.</ResponseField>
  </Expandable>
</ResponseField>
<ResponseField name="createdAt" type="string">Marca de tiempo de creación (ISO 8601).</ResponseField>
<ResponseField name="updatedAt" type="string">Marca de tiempo de última actualización (ISO 8601).</ResponseField>

### Estados del payout (`data.state`)

| Estado | Significado |
|--------|-------------|
| `created` | Payout creado |
| `in_review` | En revisión (pre-partner) |
| `awaiting_funds` | Esperando fondos |
| `funds_received` | Fondos recibidos |
| `payment_submitted` | Enviado / en proceso |
| `payment_processed` | Completado exitosamente |
| `undeliverable` | No entregable |
| `returned` | Devuelto |
| `refunded` | Reembolsado |
| `canceled` | Cancelado |
| `error` | Error / fallido |
